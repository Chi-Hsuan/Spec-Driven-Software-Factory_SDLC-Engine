# User Flow — Thinking Prompt

## Thinking Strategy

1. 針對每個 FR-ID，先問「使用者從哪裡開始、想在哪裡結束」，定義 Start 與 End。
2. 把中間過程拆成一步步的動作，每步只做一件事，標註是使用者行為還是系統回應。
3. 想像「一切順利」的情境，寫成成功路徑（Happy Path）。
4. 再想像「哪裡可能出錯」，寫成失敗路徑（Error Path），並標註觸發條件。
5. 若某一步存在多種可能結果，拆成分支條件，而不是硬塞進單一直線流程。

---

## Reasoning Rules

- **單一職責步驟**：每個 step 只描述一件事，不可把多個動作合併成一步。
- **行為 vs 系統回應二分**：每個 step 必須明確標註「使用者做了什麼」或「系統回應了什麼」，不可模糊。
- **不腦補畫面**：本階段只描述行為順序，不決定畫面長什麼樣（那是 write-wireframe 的職責）。
- **完整性**：一個 Flow 若沒有失敗路徑，需先確認是否真的不會出錯，而非省略不寫。

---

## Quality Checklist

- [ ] 每個 Flow 是否都有清楚的 Start 與 End？
- [ ] 是否每一步都標註了行為主體（使用者 or 系統）？
- [ ] 成功路徑與失敗路徑是否都已涵蓋？
- [ ] 是否混入了畫面佈局或視覺細節？
